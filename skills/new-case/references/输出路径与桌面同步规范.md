## 一、文件输出路径规范

### 1.1 标准输出根目录

所有案件相关文件统一输出至：
```
/mnt/e/律师事务部/诉讼案件/[案件名]/
```

### 1.2 按文件类型分类存储

| 文件类型 | 存放路径 | 命名规范 |
|---|---|---|
| 主档案 | `[案件名]/01_案件信息.md` | 01_案件信息.md |
| 时间轴 | `[案件名]/02_时间轴.md` | 02_时间轴.md |
| **法律文书（核心）** | `[案件名]/06 - 📝 法律文书/` | 见下方命名规范 |
| 检视清单 | `[案件名]/04_检视清单.md` | 04_检视清单.md |
| 应诉方案 | `[案件名]/02 - 📄 案件分析/` | 应诉方案.md |
| 提醒记录 | `[案件名]/00 - 📅 日程管理/` | 期限提醒.md |
| 工作日志 | `[案件名]/03_工作日志/标准化阅卷报告.md` | 标准化阅卷报告.md |
| 原始材料 | `[案件名]/04 - 📤 客户提供/` | [材料类型]_[日期] |

### 1.3 法律文书命名规范（06 - 📝 法律文书/）

```
[案件名]_[文书类型]_[版本]_[YYYYMMDD].docx

示例：
杨立发_深度分析报告_内部版_20260518.docx
杨立发_庭审复盘报告_20260518.docx
沈美娟_代理词_被告视角_CASE_FULL_LIFE_20260513.docx
周建_起诉状分析_内部版_20260518.docx
左亚青_上诉状全文_20260506.docx
```

### 1.4 桌面同步路径

桌面路径统一为（WSL 访问 Windows 桌面）：
```
/mnt/c/Users/Administrator/Desktop/
```

## 二、桌面同步触发规则

### 2.1 强制同步（自动执行）

以下情况在完成生成后**自动同步至桌面**，无需用户指令：

| 操作 | 同步文件 |
|---|---|
| 新案登记（case-registration） | 01_案件信息.md、02_时间轴.md、04_检视清单.md → .docx |
| litigation-analysis 分析完成 | 对应 .docx 文件 |
| 判决书送达 | 判决书分析报告.docx |

### 2.2 桌面文件命名规范

桌面文件名与案件目录内文件保持**完全一致**（方便快速定位），格式：
```
[案件名]_[文书类型]_[版本]_[YYYYMMDD].docx
```

示例：
```
沈美娟 vs 海安尚鸿酒店侵权纠纷_应诉策略报告_CASE_FULL_LIFE_20260513.docx
杨立发_深度分析报告_内部版_20260518.docx
```

## 三、桌面同步技术实现

### 3.1 同步命令（Python实现）

在 `register_case.py` 和 `update_status.py` 中，文件生成后自动执行：

```python
import subprocess

def sync_to_desktop(file_path, file_name=None):
    """将文件同步至Windows桌面"""
    if file_name is None:
        file_name = os.path.basename(file_path)
    desktop = "/mnt/c/Users/Administrator/Desktop/"
    cmd = f"cp '{file_path}' '{desktop}{file_name}'"
    result = subprocess.run(cmd, shell=True, capture_output=True)
    return result.returncode == 0
```

### 3.2 桌面同步的日志记录

每次桌面同步后，在 `01_案件信息.md` 的"案件全过程日志"中追加：
```
YYYY-MM-DD | 桌面同步 | [文件名].docx → C:\Users\Administrator\Desktop\
```

## 四、Litigation-analysis 输出规范化（优化五重点）

### 4.1 三场景输出路径

| 场景 | 输出文件 | 路径 |
|---|---|---|
| 判决书分析 | `[案件名]_深度分析报告_内部版_YYYYMMDD.docx` | 06 - 📝 法律文书/ |
| 起诉状分析 | `[案件名]_起诉状分析_内部版_YYYYMMDD.docx` | 06 - 📝 法律文书/ |
| 庭审复盘 | `[案件名]_庭审复盘报告_YYYYMMDD.docx` | 06 - 📝 法律文书/ |

### 4.2 桌面同步触发时机

**触发时机：每次 litigation-analysis 分析完成，立即同步至桌面**

**当前 litigation-analysis 不自动同步桌面**（技能限制），需人工执行 `cp` 命令。
优化方向：建议用户在使用 litigation-analysis 后，回复"桌面同步"，或在下一次 SKILL.md 更新时将桌面同步写入分析流程。

### 4.3 Litigation-analysis → CASE_FULL_LIFE 流转

```
litigation-analysis 生成报告
    ↓
移动/保存至 06 - 📝 法律文书/
    ↓
桌面同步 cp 命令
    ↓
更新 03_工作日志/标准化阅卷报告.md（如为庭审复盘）
    ↓
更新 01_案件信息.md 的案件全过程日志
    ↓
（如为结案归档）→ 触发 update_status.py → 知识库同步
```
