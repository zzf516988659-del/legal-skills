# 变更历史

## [0.3.0] - 2026-05-18

### 改进
- 重构步骤1"文档类型判断"的决策逻辑，解决"法律服务建议书"与"诉讼/非诉服务方案"的误匹配问题
- 新增优先级规则：案件性质（诉讼/非诉）为首要判断因素，用户措辞（"方案"≠"建议书"）为第二道防线
- 新增禁止规则：用户说"方案"时禁止自动降级为"建议书"，"建议书"仅在用户明确要求时生成
- 更新决策表排序：诉讼方案和非诉方案提到最前，建议书降至第三行并标注"用户明确要求"作为触发条件
- 更新关键判断因素：案件性质提升为首要因素，新增"用户措辞"判断维度

### 背景
实际使用中发现，当用户说"生成法律服务方案"时，AI反复匹配到"法律服务建议书"模板。根因是三个对外方案模板（诉讼服务方案、非诉服务方案、法律服务建议书）在命名和决策标准上存在重叠——"法律服务建议书"的名称包含"法律服务"，与用户通用说法"法律服务方案"词面最近，导致优先命中。而三者真正的区分维度是：业务类型（诉讼/非诉）× 客户场景（争取签约/已委托服务），建议书不应与两个方案模板平级竞争，而应作为仅限明确触发的独立文档类型。

## [0.2.0] - 2026-04-10

### 新增
- 新增“结案汇报”文档类型，用于案件以判决、调解、和解、撤诉等方式结案后向客户输出结果汇报
- 新增 `templates/case-closing-report.md`，沉淀授权条件对比、执行情况、客户实际支出和后续提醒等模块
- 新增技能级 `TASKS.md` 与 `DECISIONS.md`，补齐后续维护所需协作文档

### 改进
- 更新 `SKILL.md` 的文档类型清单、适用场景和选择逻辑，将结案收口场景纳入统一入口
- 更新 `references/modules.md`，新增“结案收口模块”和文档类型映射

### 文档完善
- 更新根目录 `README.md` 与 `.claude-plugin/marketplace.json` 中的技能描述与版本号，反映结案汇报能力已并入

### 待办事项

- 评估是否需要为执行结案、撤诉结案增加更细的可选模块
